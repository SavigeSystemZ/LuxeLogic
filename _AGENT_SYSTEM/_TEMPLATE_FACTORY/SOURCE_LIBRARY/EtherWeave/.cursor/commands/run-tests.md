# Run Tests

Run the project's tests and report results.

**Do this now:**
1. Run the project test suite. Prefer: `pytest tests/` or the command documented in the repo (e.g. `tools/verify.sh`). If no test runner is specified, run `pytest tests/ -v` or equivalent.
2. Include the **exact command** you ran and the **full output** (or a clear summary if very long).
3. If tests fail: summarize failures and recommend next steps (e.g. run **debug** command or follow `docs/DEBUG_REPAIR_PLAYBOOK.md`). Do not claim tests passed without showing output.
4. If no tests exist or the command fails to run, say so and suggest adding tests or checking the project's test docs.

Reference: docs/KERNEL_CONTRACT.md (Definition of Done), AGENTS.md (Verification Gates).
